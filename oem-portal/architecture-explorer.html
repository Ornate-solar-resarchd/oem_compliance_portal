Data Flow

How Data Moves Through the System

The three core workflows that drive the portal.

1. Datasheet Upload & AI Extraction

1

Upload PDF or Excel

User uploads OEM datasheet through the portal or fetches from Google Drive

2

Text Extraction

PyPDF2 extracts text from PDFs, openpyxl parses Excel files

3

AI Analyzes Content

Gemini (or Claude, or keyword regex) extracts 30-50+ technical specs with confidence scores

4

Compliance Scoring

Each parameter is evaluated as Pass, Fail, or Waived. Overall compliance score is calculated.

2. RFQ Processing & Matching

1

Upload Customer RFQ

Customer's technical requirements document (PDF/Excel/CSV) is uploaded

2

AI Extracts Requirements

60+ BESS parameters extracted: capacity, chemistry, certifications, grid specs, O&M terms

3

Metadata Detection

Project info auto-detected: capacity (MW/MWh), location, design life, solar integration

4

Match Against Models

Each requirement compared to all OEM models. Best-fit models ranked by match percentage.

3. Compliance Sheet & Approval

1

Create Project

Define project: name, client, type (Utility/BTM/C&I/Hybrid), capacity, location

2

Add Compliance Sheet

Link a battery model to the project. System pulls all technical parameters automatically.

3

7-Stage Approval

Sheet moves through: Draft → Engineering → Tech Lead → Management → Customer → Locked

4

Approved & Locked

Final approval locks the sheet. Compliance score and all parameters are finalized.

AI Engine

How AI Extraction Works

A 3-tier fallback system ensures reliable extraction regardless of API availability.

Extraction Priority Chain

Priority 1
Google Gemini 2.0 Flash
Free tier, 1M token context. Handles most documents. Returns structured JSON of extracted parameters.
→
Priority 2
Anthropic Claude
Paid API fallback. Activated when Gemini is unavailable or fails. Higher accuracy for complex documents.
→
Priority 3
Keyword Regex
Offline fallback. 100+ regex patterns match MW, MWh, certifications, temperature ranges, etc. Works without any API.

Approval Workflow Stages

The 7-stage compliance approval workflow powers the portal's lifecycle management.

Compliance Sheet Approval (7 Stages)

1
Draft
Initial creation
2
Engineering
Technical review
3
Tech Lead
Lead approval
4
Management
Business sign-off
5
Customer Sub
Sent to client
6
Customer OK
Client approved
7
Locked
Finalized
At each stage, reviewers can approve (advance to next), reject (return to draft), or add comments. Each sheet tracks pass/fail/waived parameter counts and overall compliance score.
